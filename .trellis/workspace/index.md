# 工作区索引

> 汇总所有开发者使用智能体开展工作的记录

---

## 概览

此目录用于记录本项目中所有开发者与智能体协作的工作情况。

### 文件结构

```
workspace/
|-- index.md              # 本文件——全局索引
+-- {developer}/          # 每位开发者的目录
    |-- index.md          # 包含会话历史的个人索引
    |-- tasks/            # 任务文件
    |   |-- *.json        # 活动任务
    |   +-- archive/      # 按月份归档的任务
    +-- journal-N.md      # 按 1、2、3……顺序编号的日志文件
```

## 开始使用

### 新开发者

运行初始化脚本：

```bash
python3 ./.trellis/scripts/init_developer.py <你的名字>
```

该脚本将：
1. 创建你的身份文件（已被 Git 忽略）
2. 创建你的工作进度目录
3. 创建你的个人索引
4. 创建初始日志文件

### 已有开发者

1. 获取你的开发者名称：
   ```bash
   python3 ./.trellis/scripts/get_developer.py
   ```

2. 读取你的个人索引：
   ```bash
   cat .trellis/workspace/$(python3 ./.trellis/scripts/get_developer.py)/index.md
   ```

## 使用规范

### 日志文件规则

- 每个日志文件**最多 2000 行**
- 达到上限后，创建 `journal-{N+1}.md`
- 创建新文件时同步更新个人 `index.md`

### 会话记录格式

每次会话应包含：
- 摘要：单行说明
- 分支：本次工作所在的分支
- 主要变更：修改了什么
- Git 提交：提交哈希和提交信息
- 后续步骤：接下来要做什么

---

## 会话模板

记录会话时使用以下模板：

```markdown
## 会话 {N}：{标题}

**日期**：YYYY-MM-DD
**任务**：{任务名称}
**分支**：`{分支名称}`

### 摘要

{单行摘要}

### 主要变更

- {变更 1}
- {变更 2}

### Git 提交

| 哈希 | 提交信息 |
|------|---------|
| `abc1234` | {提交信息} |

### 测试

- [OK] {测试结果}

### 状态

[OK] **已完成** / # **进行中** / [P] **已阻塞**

### 后续步骤

- {后续步骤 1}
- {后续步骤 2}
```

---

**语言**：所有文档必须使用**中文**。
